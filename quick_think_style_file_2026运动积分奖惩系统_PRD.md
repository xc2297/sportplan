## 1. 项目概述
### 1.1 项目背景
为帮助用户（尤其是有共同运动目标的伴侣、朋友或家庭）建立持续、有趣的运动习惯，设计一套量化激励系统。通过游戏化的积分规则、阶梯式奖励和隐藏彩蛋，提升运动乐趣与粘性。

### 1.2 核心目标
- **量化运动成果**：将不同类型的运动转化为统一积分。
- **提供即时反馈**：通过积分累积和兑换获得正向激励。
- **增强互动趣味**：通过隐藏机制、终极大奖等设计，增加用户间的互动与期待。

### 1.3 目标用户
- 有明确运动目标但缺乏动力的个人。
- 希望互相监督、共同进步的情侣、好友或家庭成员。

## 2. 功能需求
### 2.1 用户与账号管理
- 支持用户注册账号（账号 + 密码 + 姓名 + 性别）。
- 支持用户登录/登出，基于 Session 管理登录态。
- 未登录用户无法访问系统功能，自动跳转登录页。
- 用户需选择性别，以应用不同的积分系数。
- **管理员角色**：用户表含 `is_admin` 字段，管理员拥有奖励物品管理权限。

### 2.2 运动记录与积分计算
#### 2.2.1 记录方式
- 每次记录单项运动：选择运动类型 → 填写运动量 → 提交。
- 运动类型和积分系数由管理员后台动态维护，用户端仅展示已启用的类型。
- （远期）支持接入主流运动App（如Keep、悦跑圈）同步数据。

#### 2.2.2 积分规则
- 积分系数由管理员配置在运动类型表中，支持按性别差异化设置。
- 每项运动可配置单项每日积分上限（默认10分）。
- 每日总积分上限为40分。
- 积分永久有效，不清零。
- 同一用户同一日期同一运动类型仅一条记录，重复提交视为更新，差额自动调整积分账户。

#### 2.2.3 运动类型管理（管理员）
- 管理员可新增/编辑/删除/停用运动类型。
- 每个运动类型包含：名称、单位、男性系数、女性系数、单项上限、排序、启用状态。
- 系统首次启动自动初始化默认运动类型（跑步、走路、俯卧撑、仰卧起坐）。

#### 2.2.4 默认运动类型
| 运动项目 | 单位 | 女性系数 | 男性系数 | 单项上限 |
| :--- | :--- | :--- | :--- | :--- |
| 跑步 | 公里 | 1.0 | 0.5 | 10 分 |
| 走路 | 公里 | 0.5 | 0 | 10 分 |
| 俯卧撑 | 个 | 0.1 | 0.1 | 10 分 |
| 仰卧起坐 | 个 | 0.1 | 0.05 | 10 分 |

### 2.3 奖励兑换系统
#### 2.3.1 奖励管理
- 奖励物品由管理员通过后台维护页面动态管理（新增/编辑/删除）。
- 每个奖励项包含：名称、描述、所需积分、额度上限、排序权重。
- 系统首次启动时自动初始化4个默认奖励。

#### 2.3.2 默认奖励阶梯
| 消耗积分 | 奖励名称 | 内容/额度 | 说明 |
| :--- | :--- | :--- | :--- |
| 5 分 | 即时畅饮券 | ≤ 30 元 | 兑换一杯奶茶/咖啡等饮品 |
| 10 分 | 欢乐聚餐券 | ≤ 120 元 | 用于一次朋友或家庭聚餐 |
| 20 分 | 休闲娱乐券 | ≤ 200 元 | 用于电影、下午茶、游乐项目等 |
| 30 分 | 豪华大餐券 | ≤ 300 元 | 用于一次高品质餐厅体验 |

#### 2.3.2 隐藏机制
- **兑换条件**：消耗 30 积分。
- **机制内容**：兑换者可向任一其他用户提出一个**必须被满足的合理要求**。
- **限制条款**：
    1.  要求不得违背接收方的个人意愿、道德及法律。
    2.  任务需在**24小时**内可完成。
    3.  不得涉及大额金钱索取（如超过500元）。
- **仲裁条款**：若接收方认为要求不合理，双方可协商，将奖励更换为等值的"300元豪华大餐券"。

#### 2.3.3 终极大奖
| 消耗积分 | 奖励名称 | 内容说明 |
| :--- | :--- | :--- |
| 100 分 | 短途旅行计划权 | 可主导规划并执行一次**5天以内**的短途出游。团队需共同商定目的地与预算。 |
| 300 分 | 梦想旅行计划权 | 可主导规划并执行一次**不设天数与地点限制**的出游。此为年度终极奖励。 |

### 2.4 惩罚机制
- **触发条件**：未完成当周预设的积分目标。
- **惩罚选项**（可多选）：
    1.  **经济惩罚**：向"梦想基金"存入 20 元。
    2.  **运动补偿**：在周末补足所缺积分 **1.5 倍** 的运动量。
    3.  **劳务惩罚**：承包下一周某项指定家务。

### 2.5 数据统计与可视化
- **仪表盘**：展示今日积分、本周进度、累计积分、可用积分、本周每日积分柱状图。
- **积分排行榜**：全局积分排行榜，按累计积分降序展示所有用户。
- **历史记录**：按月查看运动记录和兑换记录明细。

## 4. 数据与规则定义
### 4.1 核心计算公式
```javascript
// 每日单项得分计算（伪代码）
function calculateDailyScore(gender, sportType, amount) {
  let coefficient = getCoefficient(gender, sportType); // 获取系数
  let rawScore = amount * coefficient;
  let cappedScore = Math.min(rawScore, 10); // 单项每日上限10分
  return cappedScore;
}

// 每日总分计算
dailyTotal = runningScore + walkingScore + pushupScore + situpScore; // 理论最高40分
```

## 5. 用户界面原型
### 5.1 核心页面清单
1.  **登录页**：账号密码输入，支持跳转注册。
2.  **注册页**：账号、密码、确认密码、姓名、性别表单。
3.  **首页/仪表盘**：今日积分、本周积分、累计积分、可用积分、本周每日积分图、积分排行榜。
4.  **运动记录页**：选择运动类型 → 填写运动量 → 提交，实时预估积分。已记录条目列表。管理员可见"管理运动类型"入口。
5.  **兑换中心**：动态加载奖励卡片 + 兑换确认弹窗 + 兑换记录列表。管理员可见"管理奖励"入口。
6.  **运动类型管理页（管理员）**：运动类型列表 + 新增/编辑/删除/停用，含积分系数配置。
7.  **奖励管理页（管理员）**：奖励项列表 + 新增/编辑/删除操作。
8.  **个人中心**：用户信息展示 + 运动记录日历视图（按月展示每天总分，支持翻月，点击日期查看明细）/兑换记录 + 退出登录。管理员日历显示所有用户当天积分对比，点击查看多用户详情对比。

### 5.2 导航结构
- 底部 4 个 Tab：首页、记录、兑换、我的
- 已登录状态下显示底部导航栏，未登录时隐藏

### 5.3 关键交互描述
- **注册/登录**：首次使用先注册，注册成功跳转登录页；登录成功跳转首页；刷新页面自动恢复登录态。
- **记录运动**：选择日期和运动类型，填写运动量后，实时显示预估积分，提交后自动计算并更新积分。同一类型重复提交视为更新。
- **兑换奖励**：在兑换中心点击奖励卡，弹出确认模态框，兑换后积分实时扣除。
- **退出登录**：在个人中心点击"退出登录"，清除登录态，跳转登录页。
- **运动日历**：个人中心运动记录以日历格式展示，每格显示当天总分（普通用户）或各用户分数（管理员，如"张三:20 李四:30"）。支持翻月查看历史。点击有数据的日期弹出详情弹窗，显示各项运动明细；管理员显示多用户对比卡片。

## 6. 运营与风控
| 风险 | 可能性 | 影响 | 应对措施 |
| :--- | :--- | :--- | :--- |
| **刷分作弊** | 中 | 高 | 设置每日上限；未来可接入运动设备数据验证。 |
| **奖励成本失控** | 低 | 中 | 奖励多为"兑换券"，本质是消费许可，成本由用户自行承担。可设定月度上限。 |
| **隐藏机制纠纷** | 中 | 低 | 明确条款，设立"可协商并降级为等值大餐券"的仲裁机制。 |
| **用户流失** | 中 | 中 | 通过推送提醒、定期更新奖励、增加节日活动保持新鲜感。 |
